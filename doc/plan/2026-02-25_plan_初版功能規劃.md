# 初版功能規劃

**日期**：2026-02-25
**狀態**：已完成實作

---

## 背景

LINE 官方帳號每月有免費訊息額度，超過後依方案計費。目前需要人工查看 LINE OA Manager 後台，缺乏自動化通知機制。

## 目標

建立自動化服務，每月自動計算 LINE 訊息用量並估算加購費用，推播到指定 LINE 群組。

---

## 核心功能

### 1. 每日快照（Daily Snapshot）

- 觸發時間：每日 23:55 Asia/Taipei
- 呼叫 LINE Messaging API 取得當月累計用量
- 存入 DynamoDB `usage_snapshots` 資料表
- 幂等性設計：同一天重複觸發不寫入

### 2. 跨月封存（Month-end Archive）

- 月份變更時自動標記上月最後一筆快照為 `isPrevMonthFinal = true`
- 支援補封存：若月初才部署，可手動觸發補標記

### 3. 每月回報（Monthly Report）

- 觸發時間：每月 11 日 09:00 Asia/Taipei（LINE 帳單約 10 日產出）
- 讀取上月 `prevMonthFinal` 快照
- 計算加購費用（支援 single / tiers 兩種計費模型）
- 推播繁中格式訊息至指定 LINE 群組

---

## 計費模型

### 費用組成

```
每月總費用 = 方案月費（PLAN_FEE）+ 加購費用
```

`PLAN_FEE` 設為 `0` 時僅顯示加購費用，不顯示合計欄位。

### single 模式

每則超額訊息固定單價，由 `SINGLE_UNIT_PRICE` 設定。

### tiers 模式（累進級距）

由 `TIERS_JSON` 設定級距，費用依各級距累進計算，最後一級 `upTo: null` 表示無上限。

```json
[
  { "upTo": 10000, "price": 0.2 },
  { "upTo": 50000, "price": 0.18 },
  { "upTo": null,  "price": 0.16 }
]
```

## 技術選型

| 元件 | 選擇 | 理由 |
|------|------|------|
| 運算 | AWS ECS Fargate | 見 [ADR-001](../adr/2026-02-25_adr-001_選用ECS-Fargate而非Lambda.md) |
| 排程 | EventBridge Scheduler | cron 表達式支援時區，穩定度高 |
| 儲存 | DynamoDB | 低成本、免維護、適合 key-value 快照存取 |
| 機密管理 | SSM Parameter Store | 與 ECS Task 原生整合 |
| 日誌 | CloudWatch Logs + pino | JSON 結構化日誌，便於 Insights 查詢 |
| 語言 | Node.js | 輕量，LINE SDK 生態完整 |

---

## 部署方式

- Infrastructure as Code：AWS CDK（TypeScript）
- CI/CD：GitHub Actions（push `v*` tag 觸發）
- Image 版本：禁用 `latest`，強制使用明確版本 tag（如 `v20260225-1`）

---

## 已知限制

- LINE consumption API 回傳的 `totalUsage` 為**近似值**，最終帳單請以 LINE OA Manager 後台為準
- 目前僅支援單一 LINE 官方帳號
