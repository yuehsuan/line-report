# 修復計畫：ECS Task 無法讀取 LINE_CHANNEL_ACCESS_TOKEN

**日期**：2026-03-03  
**狀態**：待批准  
**影響範圍**：`iac/lib/ecs-stack.ts`、AWS SSM Parameter Store  

---

## 一、問題描述

執行 ECS Fargate task（`snapshot` / `report` 指令）時，容器啟動後立即以 `EssentialContainerExited` 停止，  
CloudWatch Logs 顯示以下錯誤：

```
"level":"error"
"message":"環境變數 LINE_CHANNEL_ACCESS_TOKEN 未設定"
"msg":"LINE API 網路錯誤 [getConsumption]"
```

---

## 二、根本原因分析

### 原因 A：SSM 參數值為佔位符字串（直接致命原因）

透過 `aws ssm get-parameters-by-path --path /line-report/` 查詢結果：

| 參數名稱 | 實際值 |
|---|---|
| `/line-report/LINE_CHANNEL_ACCESS_TOKEN` | `（貼上 .env 的 LINE_CHANNEL_ACCESS_TOKEN）` |
| `/line-report/LINE_TARGETS` | `U52dbab066ad1e272ccb30489213ad8a9,...` ✅ |

`LINE_CHANNEL_ACCESS_TOKEN` 的值是一段**中文說明文字**，不是真正的 token。  
推測是當初部署後手動 `put-parameter` 時，直接把指引說明文字貼進去，而非貼上真實 token。

### 原因 B：`ecs-stack.ts` 沒有將 SSM 值注入為容器環境變數（結構缺漏）

`ecs-stack.ts` 的 `addContainer` 只定義了 5 個靜態環境變數：

```typescript
environment: {
  TZ: 'Asia/Taipei',
  AWS_REGION: this.region,
  DDB_TABLE_SNAPSHOTS: snapshotsTable.tableName,
  DDB_TABLE_RUNS: runsTable.tableName,
  LOG_LEVEL: 'info',
},
```

`LINE_CHANNEL_ACCESS_TOKEN`、`LINE_TARGETS`、`FREE_QUOTA`、`PRICING_MODEL`、`PLAN_FEE`、`SINGLE_UNIT_PRICE`、`CURRENCY` **均未注入**。

而 `src/lib/lineApi.js` 讀取的是 `process.env.LINE_CHANNEL_ACCESS_TOKEN`（直接讀環境變數），  
並非主動呼叫 AWS SDK 去查詢 SSM，因此 ECS task role 上既有的 `ssm:GetParameter` 權限在此情境下無法解決問題。

### 兩個原因的關係

| 層面 | 問題 | 說明 |
|---|---|---|
| 資料面 | SSM 值是佔位符 | 即使 ECS 有注入機制，注入的也是錯誤值 |
| 架構面 | Task Definition 沒有 `secrets` 注入 | 即使 SSM 值正確，容器也看不到 |

**兩者必須同時修復才能讓 task 正常執行。**

---

## 三、修復計畫

### Step 1：修正 SSM 參數值

將 `.env` 中的真實 token 寫入 SSM。執行以下 CLI 指令：

```bash
# 1-1. 更新 LINE_CHANNEL_ACCESS_TOKEN（SecureString，加密儲存）
aws ssm put-parameter \
  --profile srec \
  --name /line-report/LINE_CHANNEL_ACCESS_TOKEN \
  --type SecureString \
  --value "<從 .env 複製真實 token>" \
  --overwrite

# 1-2. 確認無誤
aws ssm get-parameter \
  --profile srec \
  --name /line-report/LINE_CHANNEL_ACCESS_TOKEN \
  --with-decryption \
  --query 'Parameter.Value'
```

> `LINE_TARGETS` 已有正確值，不需更新。

---

### Step 2：修改 `iac/lib/ecs-stack.ts`，加入 SSM secrets 注入

#### 2-1. 給 Execution Role 加 SSM 讀取權限

ECS 使用 `secrets` 欄位注入時，是由 **Execution Role**（不是 Task Role）在容器啟動前去 SSM 拉值。  
目前 Execution Role 只有 `AmazonECSTaskExecutionRolePolicy`（涵蓋 ECR + CloudWatch），沒有 SSM 權限，  
因此需要額外加上：

```typescript
// 新增於 executionRole 建立後
executionRole.addToPolicy(new iam.PolicyStatement({
  sid: 'SsmSecretsAccess',
  effect: iam.Effect.ALLOW,
  actions: [
    'ssm:GetParameters',
  ],
  resources: [
    `arn:aws:ssm:${this.region}:${this.account}:parameter/line-report/*`,
  ],
}));
```

#### 2-2. 在 Container Definition 加入 `secrets` 欄位

```typescript
this.taskDefinition.addContainer('app', {
  // ... 現有設定不變 ...
  environment: {
    TZ: 'Asia/Taipei',
    AWS_REGION: this.region,
    DDB_TABLE_SNAPSHOTS: snapshotsTable.tableName,
    DDB_TABLE_RUNS: runsTable.tableName,
    LOG_LEVEL: 'info',
  },
  // ↓ 新增：從 SSM 注入機密與業務設定
  secrets: {
    LINE_CHANNEL_ACCESS_TOKEN: ecs.Secret.fromSsm(
      ssm.StringParameter.fromSecureStringParameterAttributes(this, 'SsmToken', {
        parameterName: '/line-report/LINE_CHANNEL_ACCESS_TOKEN',
      })
    ),
    LINE_TARGETS: ecs.Secret.fromSsm(
      ssm.StringParameter.fromStringParameterName(this, 'SsmTargets',
        '/line-report/LINE_TARGETS'
      )
    ),
    FREE_QUOTA: ecs.Secret.fromSsm(
      ssm.StringParameter.fromStringParameterName(this, 'SsmFreeQuota',
        '/line-report/FREE_QUOTA'
      )
    ),
    PRICING_MODEL: ecs.Secret.fromSsm(
      ssm.StringParameter.fromStringParameterName(this, 'SsmPricingModel',
        '/line-report/PRICING_MODEL'
      )
    ),
    PLAN_FEE: ecs.Secret.fromSsm(
      ssm.StringParameter.fromStringParameterName(this, 'SsmPlanFee',
        '/line-report/PLAN_FEE'
      )
    ),
    SINGLE_UNIT_PRICE: ecs.Secret.fromSsm(
      ssm.StringParameter.fromStringParameterName(this, 'SsmSingleUnitPrice',
        '/line-report/SINGLE_UNIT_PRICE'
      )
    ),
    CURRENCY: ecs.Secret.fromSsm(
      ssm.StringParameter.fromStringParameterName(this, 'SsmCurrency',
        '/line-report/CURRENCY'
      )
    ),
  },
  // ...
});
```

需在檔案頂端新增 SSM import：
```typescript
import * as ssm from 'aws-cdk-lib/aws-ssm';
```

### Step 3：重新部署 CDK（僅 ECS Stack）

```bash
npm run deploy -- LineReportEcsStack
```

> 只有 Task Definition 需要更新，不需重建 Docker image。

### Step 4：驗證

部署完成後，手動觸發一次 snapshot task：

```bash
aws ecs run-task \
  --profile srec \
  --cluster line-report \
  --task-definition line-report \
  --launch-type FARGATE \
  --network-configuration "awsvpcConfiguration={subnets=[subnet-06eb77ca7b4b4df06],securityGroups=[sg-085e167d9f0748a1e],assignPublicIp=ENABLED}" \
  --overrides '{"containerOverrides":[{"name":"app","command":["node","src/index.js","snapshot"]}]}'
```

確認以下項目：
1. Task 狀態為 `STOPPED`，StopCode 為 `EssentialContainerExited`（正常結束）而非因錯誤停止
2. CloudWatch Logs 有「快照執行完成」的 `info` 訊息，無 `error`
3. DynamoDB `usage_snapshots` 資料表出現 `2026-03` 的新紀錄
